    Driftnet OSINT Endpoints¶

    The Driftnet endpoints provide comprehensive internet intelligence via the Driftnet.io API. With 25 endpoints spanning DNS, WHOIS, IP intelligence, scanning, certificate transparency, and vulnerability data, Driftnet offers deep OSINT capabilities for security research and threat intelligence.

    Configuration¶

    API Key Required

    All Driftnet endpoints require a DRIFTNET_API_KEY to be configured.

    Environment Variable
    # Add to .env or environment variables
DRIFTNET_API_KEY=your_api_key_here

    Base URL

    All endpoints use the base URL http://localhost:8001.

    Endpoint Categories¶

    Category Endpoints Description
    DNS 4 Forward/reverse DNS, MX, TXT records
    Domain 3 WHOIS, reverse WHOIS, comprehensive intel
    IP 7 WHOIS, BGP, abuse, routes, reverse lookup
    Scanning 5 Protocol scans, ports, JARM, favicons
    Certificate Transparency 2 CT log search, subdomain enumeration
    Multi-Source 1 Cross-source correlation
    Reference 1 CVE lookups
    Suggestions 2 Autocomplete for domains and scan values

    DNS Endpoints¶

    GET /v1/driftnet/dns/forward¶

    Forward DNS lookups (A/AAAA records) with historical data support.

    Best For

    Finding IP addresses for hostnames, tracking DNS history over time, and correlating hostnames to IPs.

    Parameters:

    Name Type Description
    host string Hostname to lookup
    ip string IP address or CIDR to search
    nameserver string Filter by nameserver
    summarize string Field to summarize (e.g., 'ip', 'host')
    from string Start date (YYYY-MM-DD)
    to string End date (YYYY-MM-DD)
    page integer Result page (0-indexed) 
    # Find all IPs that have hosted google.com
curl "http://localhost:8001/v1/driftnet/dns/forward?host=google.com"

# Find all domains resolving to a specific IP
curl "http://localhost:8001/v1/driftnet/dns/forward?ip=8.8.8.8"

# Summarize unique IPs for a domain
curl "http://localhost:8001/v1/driftnet/dns/forward?host=example.com&summarize=ip"

    Response:

    {
  "success": true,
  "data": {
    "page": 0,
    "pages": 134,
    "result_count": 13330,
    "results": [
      {
        "date": "2026-02-02",
        "items": [
          {"type": "host", "value": "mail.google.com"},
          {"type": "ip", "value": "209.85.128.238", "context": "dns-a"},
          {"type": "host", "value": "ns4.google.com", "context": "dns-ns"}
        ]
      }
    ]
  }
}

    Response Fields:

    Field Type Description
    page integer Current page number (0-indexed)
    pages integer Total number of pages
    result_count integer Total result count
    results[].date string Date of the DNS record observation
    results[].items array DNS record items with type, value, and context

    GET /v1/driftnet/dns/txt¶

    TXT record lookups (SPF, DKIM, DMARC, and other verification records).

    Best For

    Analyzing email security configuration, checking SPF/DKIM/DMARC posture, and finding domain verification tokens.

    Parameters:

    Name Type Description
    host string Domain to lookup TXT records
    txt string TXT content to search (e.g., 'v=spf1')
    prefix boolean Treat txt as prefix match
    from string Start date
    to string End date
    page integer Result page 
    curl "http://localhost:8001/v1/driftnet/dns/txt?host=google.com"

    # Find all domains with specific SPF configuration
curl "http://localhost:8001/v1/driftnet/dns/txt?txt=v=spf1&prefix=true"

    Domain Endpoints¶

    GET /v1/driftnet/domain/whois¶

    Domain WHOIS lookup returning structured registration data.

    Best For

    Researching domain ownership, registration dates, registrar information, and brand protection.

    Parameters:

    Name Type Description
    domain string Domain name to lookup (required)
    extract_apex boolean Extract apex domain from subdomain 
    curl "http://localhost:8001/v1/driftnet/domain/whois?domain=google.com"

    # Extract apex domain from subdomain
curl "http://localhost:8001/v1/driftnet/domain/whois?domain=mail.google.com&extract_apex=true"

    Response:

    {
  "success": true,
  "data": {
    "domain": {
      "name": "google.com",
      "created": "1997-09-15T04:00:00Z",
      "expires": "2028-09-14T04:00:00Z",
      "registrar_name": "MarkMonitor Inc.",
      "nameservers": ["ns1.google.com", "ns2.google.com", "ns3.google.com", "ns4.google.com"],
      "status": "clientDeleteProhibited,clientTransferProhibited,clientUpdateProhibited"
    }
  }
}

    Response Fields:

    Field Type Description
    domain.name string Domain name
    domain.created string Registration date (ISO 8601)
    domain.expires string Expiration date (ISO 8601)
    domain.registrar_name string Registrar name
    domain.nameservers array Authoritative nameservers
    domain.status string Domain status codes

    GET /v1/driftnet/domain/reverse¶

    Reverse WHOIS search -- find domains by registrant information.

    Best For

    Mapping threat actor infrastructure, finding all domains owned by an entity, and brand protection.

    Parameters:

    Name Type Description
    name string Registrant name
    email string Registrant email
    phone string Registrant phone
    nameserver string Nameserver
    phrase string Search phrase in any field
    prefix boolean Treat as prefix match
    created_from string Domain creation start date
    created_to string Domain creation end date
    page integer Result page 
    curl "http://localhost:8001/v1/driftnet/domain/reverse?name=Google"

    curl "http://localhost:8001/v1/driftnet/domain/reverse?email=admin@example.com"

    curl "http://localhost:8001/v1/driftnet/domain/reverse?nameserver=ns1.google.com"

    Scanning Endpoints¶

    GET /v1/driftnet/scan/protocols¶

    Protocol scan data for finding hosts by server software, HTTP headers, page titles, and more.

    Best For

    Finding vulnerable services, tracking C2 infrastructure, and service discovery.

    Parameters:

    Name Type Description
    ip string IP address or CIDR
    query string Free-text search
    field string Field search (e.g., 'server:nginx')
    expression string Complex query expression
    filter string Post-filter results
    most_recent boolean Dedupe to most recent
    summarize string Field to summarize
    summary_limit integer Max summary values
    from string Start date
    to string End date
    page integer Result page 
    # Find all IPs running nginx
curl "http://localhost:8001/v1/driftnet/scan/protocols?field=server:nginx"

    # Find Apache servers with specific title
curl "http://localhost:8001/v1/driftnet/scan/protocols?field=title:admin&query=apache"

    # Get scan data for specific IP
curl "http://localhost:8001/v1/driftnet/scan/protocols?ip=8.8.8.8"

    GET /v1/driftnet/scan/ports¶

    Open ports summary with honeypot detection.

    Best For

    Quick view of open ports for an IP or CIDR range.

    Parameters:

    Name Type Description

    Response:

    GET /v1/driftnet/scan/jarm¶

    JARM fingerprint lookups for tracking C2 infrastructure and identifying server implementations.

    Best For

    Tracking malware C2 servers, server fingerprinting, and infrastructure correlation. Same JARM hash indicates similar server configuration.

    Parameters:

    Name Type Description
    ip string IP address
    hash string JARM hash to search
    summarize string Field to summarize (e.g., 'ip')
    summary_limit integer Max summary values
    from string Start date
    to string End date
    page integer Result page 
    curl "http://localhost:8001/v1/driftnet/scan/jarm?ip=8.8.8.8"

    # Find all IPs with same JARM hash
curl "http://localhost:8001/v1/driftnet/scan/jarm?hash=29d29d00029d29d00042d43d00041d2aa5ce6a70de7ba95aef77a77b00a0af"

    GET /v1/driftnet/scan/favicons¶

    Favicon hash lookups for finding related infrastructure by favicon hash.

    Best For

    Tracking phishing infrastructure, finding impersonation sites, and infrastructure correlation. Same favicon likely indicates same or related infrastructure.

    Parameters:

    Name Type Description
    sha1 string SHA1 hash
    sha256 string SHA256 hash
    md5 string MD5 hash
    murmur3 integer MurmurHash3 (Shodan-compatible format)
    summarize string Field to summarize
    from string Start date
    to string End date
    page integer Result page 
    # Find IPs with same favicon (MurmurHash3 format for Shodan compatibility)
curl "http://localhost:8001/v1/driftnet/scan/favicons?murmur3=-1123257478"

    Supported Hash Formats

    Multiple hash formats are supported: SHA1, SHA256, MD5, and MurmurHash3. The MurmurHash3 format is compatible with Shodan's http.favicon.hash filter.

    Certificate Transparency Endpoints¶

    GET /v1/driftnet/ct/log¶

    Certificate Transparency log searches for finding certificates issued for domains.

    Best For

    Certificate reconnaissance, subdomain enumeration via CT logs, and certificate inventory/compliance.

    Parameters:

    Name Type Description
    field string Field search (e.g., 'host:example.com') (required)
    prefix boolean Treat as prefix match (for wildcards)
    valid_filter string 'valid-from' or 'valid-to'
    summarize string Field to summarize (e.g., 'host')
    summary_limit integer Max summary values
    from string Start date
    to string End date
    page integer Result page 
    # Find all certificates for a domain
curl "http://localhost:8001/v1/driftnet/ct/log?field=host:github.com"

    # Enumerate subdomains using CT logs
curl "http://localhost:8001/v1/driftnet/ct/log?field=host:example.com&prefix=true&summarize=host"

    # Find certificates by organization
curl "http://localhost:8001/v1/driftnet/ct/log?field=subject:Google"

    GET /v1/driftnet/subdomains¶

    Enumerate subdomains using CT logs and DNS combined for comprehensive subdomain discovery.

    Best For

    Attack surface discovery, asset inventory, scope enumeration for penetration testing and bug bounties.

    Parameters:

    Name Type Description
    domain string Apex domain to enumerate (required)
    limit integer Max subdomains (default: 1000) 
    curl "http://localhost:8001/v1/driftnet/subdomains?domain=github.com&limit=100"

    Response:

    {
  "success": true,
  "data": {
    "domain": "github.com",
    "ct_subdomains": {},
    "dns_subdomains": {}
  }
}

    Response Fields:

    Field Type Description
    domain string The queried apex domain
    ct_subdomains object Subdomains discovered from Certificate Transparency logs
    dns_subdomains object Subdomains discovered from DNS data

    Reference Data¶

    GET /v1/driftnet/cve¶

    CVE information lookup with CVSS scoring, EPSS probability, and KEV status.

    Best For

    Vulnerability research, patch prioritization, and impact assessment.

    Parameters:

    Name Type Description
    id string CVE ID (e.g., 'CVE-2021-44228') (required) 
    curl "http://localhost:8001/v1/driftnet/cve?id=CVE-2021-44228"

    curl "http://localhost:8001/v1/driftnet/cve?id=CVE-2024-1234"

    Response:

    {
  "success": true,
  "data": {
    "id": "CVE-2021-44228",
    "cvss_score": "10.0",
    "cvss_version": "3.1",
    "descriptions": {
      "en": "Apache Log4j2 2.0-beta9 through 2.15.0 JNDI features..."
    },
    "epss_score": "0.94358",
    "epss_percentile": "0.9996",
    "kev": {
      "date_added": "2021-12-10",
      "due_date": "2021-12-24",
      "known_ransomware_campaign_use": "Known"
    }
  }
}

    Response Fields:

    Field Type Description
    id string CVE identifier
    cvss_score string CVSS base score (0.0-10.0)
    cvss_version string CVSS version used for scoring
    descriptions object CVE descriptions by language
    epss_score string EPSS probability of exploitation (0-1)
    epss_percentile string EPSS percentile ranking
    kev.date_added string Date added to CISA KEV catalog
    kev.due_date string Required remediation date
    kev.known_ransomware_campaign_use string Whether used in ransomware campaigns

    Scoring Systems

    • CVSS: Common Vulnerability Scoring System -- measures severity (0-10)
    • EPSS: Exploit Prediction Scoring System -- predicts likelihood of exploitation (0-1)
    • KEV: CISA Known Exploited Vulnerabilities -- indicates active exploitation in the wild
