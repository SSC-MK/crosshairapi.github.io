    Telegram Intel¶

    Telegram threat intelligence channel tracking and OSINT capabilities. Monitor ransomware groups, data leak marketplaces, hacktivist channels, stealer log sellers, and other threat actors operating on Telegram.

    Overview

    The Telegram Intel module provides a curated database of threat-related Telegram channels with categorization, risk scoring, and threat actor attribution. Sync from external sources like ransomware.live to stay current.

    Quick Filters¶

    Specialized endpoints for the most common threat categories:

    Endpoint Description
    GET /v1/telegram/critical All critical-risk channels (highest priority)
    GET /v1/telegram/ransomware Ransomware operator channels
    GET /v1/telegram/data-leaks Data breach marketplaces
    GET /v1/telegram/stealer-logs Infostealer credential channels
    GET /v1/telegram/hacktivism Hacktivist group channels

    Channel Categories¶

    GET /v1/telegram/categories¶

    List all channel categories with counts.

    Categories include:

    Category Description
    ransomware Ransomware operator channels
    data_leaks Data breach marketplaces
    hacking_tools Hacking tools and utilities
    malware Malware samples and development
    stealer_logs Infostealer logs (RedLine, Raccoon, Vidar, etc.)
    initial_access Initial access brokers
    apt_groups APT group channels
    hacktivism Hacktivist groups 
    curl http://localhost:8001/v1/telegram/categories

    Channel Management¶

    GET /v1/telegram/channels¶

    List all tracked Telegram channels. Returns channels sorted by risk level (critical first).

    Parameters:

    Name Type Default Description
    category string - Filter by category (ransomware, data_leaks, etc.)
    risk_level string - Filter: low, medium, high, critical
    tags string - Filter by tags
    search string - Search within channel data
    threat_actor string - Filter by associated threat actor
    limit integer 100 Maximum results
    offset integer 0 Pagination offset 
    # List all ransomware channels
curl "http://localhost:8001/v1/telegram/channels?category=ransomware"

# List critical-risk channels
curl "http://localhost:8001/v1/telegram/channels?risk_level=critical&limit=20"

# Filter by threat actor
curl "http://localhost:8001/v1/telegram/channels?threat_actor=LockBit"

    import requests

# List high-risk data leak channels
response = requests.get("http://localhost:8001/v1/telegram/channels", params={
    "category": "data_leaks",
    "risk_level": "high",
    "limit": 50
})
channels = response.json()["data"]["channels"]
for ch in channels:
    print(f"@{ch['username']} - {ch['title']} [{ch['risk_level']}]")

    POST /v1/telegram/channels¶

    Add a new channel to track.

    Request Body:

    Field Type Required Description
    username string Yes Telegram username (without @)
    title string Yes Display name
    category string No Channel category
    threat_actor string No Associated threat actor/group
    risk_level string No low, medium, high, critical
    tags array No List of tags 
    curl -X POST http://localhost:8001/v1/telegram/channels \
  -H "Content-Type: application/json" \
  -d '{
    "username": "example_channel",
    "title": "Example Threat Channel",
    "category": "ransomware",
    "threat_actor": "ExampleGroup",
    "risk_level": "high",
    "tags": ["ransomware", "RaaS"]
  }'

    GET /v1/telegram/channels/search¶

    Full-text search across all channel fields (username, title, description, threat actor, tags).

    Parameters:

    Name Type Required Default Description
    q string Yes - Search query
    limit integer No 50 Maximum results 
    curl "http://localhost:8001/v1/telegram/channels/search?q=lockbit&limit=20"

    GET /v1/telegram/channels/{username}¶

    Get full details for a specific channel including category, tags, threat actor, risk level, member count, and direct URL.

    curl http://localhost:8001/v1/telegram/channels/example_channel

    PATCH /v1/telegram/channels/{username}¶

    Update a channel's metadata. All fields are optional -- only provided fields will be updated.

    curl -X PATCH http://localhost:8001/v1/telegram/channels/example_channel \
  -H "Content-Type: application/json" \
  -d '{"risk_level": "critical", "tags": ["ransomware", "RaaS", "active"]}'

    DELETE /v1/telegram/channels/{username}¶

    Remove a channel from tracking.

    curl -X DELETE http://localhost:8001/v1/telegram/channels/example_channel

    Intelligence & Analytics¶

    GET /v1/telegram/stats¶

    Overall statistics: total channels, breakdown by category, risk level, country, and tracked threat actors.

    curl http://localhost:8001/v1/telegram/stats

    GET /v1/telegram/threat-actors¶

    List all tracked threat actors with their associated channels.

    curl http://localhost:8001/v1/telegram/threat-actors

    GET /v1/telegram/sources¶

    List available external data sources for syncing.

    curl http://localhost:8001/v1/telegram/sources

    Data Management¶

    POST /v1/telegram/sync¶

    Sync channels from external threat intel sources. Currently syncs from ransomware.live Telegram data. Adds new channels and updates member counts for existing ones.

    curl -X POST http://localhost:8001/v1/telegram/sync

    GET /v1/telegram/export¶

    Export channels in various formats.

    Parameters:

    Name Type Default Description
    format string json Export format: json, csv, urls
    category string - Filter by category 
    curl "http://localhost:8001/v1/telegram/export?format=json&category=ransomware"

    curl "http://localhost:8001/v1/telegram/export?format=csv" -o channels.csv

    # Plain list of t.me URLs
curl "http://localhost:8001/v1/telegram/export?format=urls"
