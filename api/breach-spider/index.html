    Breach Spider¶

    Extract all 90,000+ breach notification records from the Data Breach Chronology database (databreachchronology.org).

    Overview¶

    The Breach Spider is a parallel extraction engine built on Camoufox anti-detect browsers. It partitions the database by state and organization type, rotates through 100 residential proxies, and deduplicates results by record UUID -- delivering the complete dataset in approximately 38 minutes with zero rate limits.

    Observed Performance

    Metric Value
    Total records 90,278
    Extraction time ~38 minutes
    API calls 974
    Rate limits hit 0
    Errors 0
    Proxy rotations 47
    Speed ~2.6 s per page (Turnstile refresh + API call)

    Key Features¶

    Feature Description
    Parallel Camoufox browsers Multiple anti-detect Firefox instances run concurrently
    100 residential proxy rotation Proactive rotation at 4K records, before the 5K rate limit
    Cloudflare Turnstile bypass page.expect_response() at the browser protocol level
    State-based partitioning 53 states + UNKN broken by 8 org types = 61 zero-overlap partitions
    Deduplication By record UUID -- no duplicates even with overlapping strategies
    Export JSON or CSV download

    Endpoints¶

    POST /v1/breaches/spider/parallel -- Start Parallel Extraction¶

    Run parallel breach extraction using Camoufox browsers. This is the recommended endpoint.

    Query Parameters¶

    Parameter Type Default Description
    strategy string states Partition strategy (see table below)
    max_records int null Maximum records to collect (null = unlimited)
    max_workers int 3 Parallel browser instances (1-20)
    proxy_file string auto Path to proxy file (auto-detects /app/proxies.txt)
    proxy_key string null Legacy Webshare API key
    background bool true Run in background (recommended)

    Partition Strategies¶

    Strategy Partitions Description
    states 61 53 states + UNKN broken by 8 org types. Recommended. Zero overlap.
    org_types 8 By organization type (BSO, BSF, MED, UNKN, BSR, EDU, NGO, GOV)
    states_and_types 385+ Full grid: state x org_type. Maximum parallelism.
    search 159 Search terms (has overlap, deduped by UUID)
    search_extended 360+ Search terms + letter pair combinations
    full 220+ States + UNKN breakdown + search terms. Maximum coverage.

    Examples¶

    curl -X POST "http://localhost:8001/v1/breaches/spider/parallel"

    curl -X POST "http://localhost:8001/v1/breaches/spider/parallel?strategy=states&max_workers=1&max_records=500"

    curl -X POST "http://localhost:8001/v1/breaches/spider/parallel?strategy=states&max_workers=5"

    Response¶

    {
  "success": true,
  "message": "Parallel spider started in background",
  "data": {
    "strategy": "states", // (1)
    "max_workers": 3, // (2)
    "proxy": "residential_file", // (3)
    "max_records": null,
    "check_status": "/v1/breaches/spider/status"
  }
}
    1. The partition strategy used to divide the database for parallel extraction. "states" splits by US state + UNKN, broken into org-type sub-partitions for zero overlap.
    2. Number of concurrent Camoufox anti-detect browser instances. Higher values speed up extraction but consume more memory (~500 MB per worker). Max 20.
    3. Proxy source in use. "residential_file" means proxies were loaded from the local file (/app/proxies.txt). Proxies rotate proactively at 4K records to stay below the 5K rate-limit threshold.

    POST /v1/breaches/spider/extract-all -- Full Extraction¶

    Convenience wrapper that runs /spider/parallel with the full strategy.

    Query Parameters¶

    Parameter Type Default Description
    max_workers int 3 Parallel browser instances
    proxy_file string auto Path to proxy file
    proxy_key string null Legacy Webshare API key

    Example¶

    curl -X POST "http://localhost:8001/v1/breaches/spider/extract-all"

    Response¶

    {
  "success": true,
  "message": "Full extraction started in background",
  "data": {
    "strategy": "full",
    "max_workers": 3,
    "proxy": "residential_file",
    "partitions": "53 states + UNKN x 7 org_types + 159 search terms",
    "target_records": "90,000+",
    "proxy_capacity": "100 proxies x 5K = 500K records",
    "check_status": "/v1/breaches/spider/status",
    "get_records": "/v1/breaches/spider/records",
    "export": "/v1/breaches/spider/export"
  }
}

    GET /v1/breaches/spider/status -- Check Status¶

    Get the current spider run status. Poll this while extraction runs in the background.

    Example¶

    curl "http://localhost:8001/v1/breaches/spider/status"

    Status Values¶

    Status Description
    never_run Spider has not been run yet
    running Extraction is in progress
    completed Extraction finished successfully
    failed Extraction encountered a fatal error

    Response (running)¶

    {
  "success": true,
  "data": {
    "status": "running",
    "started_at": "2026-02-11T17:11:03.352824",
    "strategy": "states",
    "max_workers": "3",
    "proxy": "residential_file",
    "records_collected": "0"
  }
}

    Response (completed)¶

    {
  "success": true,
  "data": {
    "status": "completed",
    "started_at": "2026-02-11T17:11:03.352824",
    "completed_at": "2026-02-11T17:48:54.355182",
    "records_collected": "90278",
    "unique_records": "90278",
    "rate_limits_hit": "0"
  }
}

    GET /v1/breaches/spider/records -- Get Records¶

    Get collected breach records with optional filtering.

    Query Parameters¶

    Parameter Type Default Description
    limit int 100 Records per page
    offset int 0 Pagination offset
    state string null Filter by state code (e.g. CA, NY, TX)
    org_type string null Filter by organization type (e.g. MED, BSO, EDU)
    search string null Search organization name (case-insensitive substring)

    Examples¶

    curl "http://localhost:8001/v1/breaches/spider/records?limit=10"

    curl "http://localhost:8001/v1/breaches/spider/records?state=CA&org_type=MED"

    Field Type Description

    organization string Normalised organization name

    breach_date string Date breach occurred

    GET /v1/breaches/spider/export -- Export Records¶

    Export all collected records as a JSON or CSV download.

    Query Parameters¶

    Parameter Type Default Description
    format string json Export format: json or csv

    CSV columns: id, organization, reported_date, breach_date, breach_type, total_affected, organization_type, location_state, source

    Examples¶

    curl "http://localhost:8001/v1/breaches/spider/export?format=csv" -o breach_records.csv

    curl "http://localhost:8001/v1/breaches/spider/export?format=json" -o breach_records.json

    DELETE /v1/breaches/spider/records -- Clear Records¶

    Clear all stored breach records, stats, and status. Use before starting a fresh extraction.

    POST /v1/breaches/spider/run (deprecated)¶

    Deprecated

    This legacy endpoint is superseded by /v1/breaches/spider/parallel. Use the parallel endpoint for all new integrations.

    Typical Workflow¶

    The recommended workflow is: clear previous data, start extraction, poll status, review stats, and export.

    # 1. Clear any previous data
curl -X DELETE "http://localhost:8001/v1/breaches/spider/records"

# 2. Start extraction
curl -X POST "http://localhost:8001/v1/breaches/spider/parallel?strategy=states"

# 3. Monitor progress (poll every 30s)
watch -n 30 'curl -s http://localhost:8001/v1/breaches/spider/status | python3 -m json.tool'

# 4. Check stats when complete
curl "http://localhost:8001/v1/breaches/spider/stats"

# 5. Export as CSV
curl "http://localhost:8001/v1/breaches/spider/export?format=csv" -o breaches.csv

# 6. Query specific records
curl "http://localhost:8001/v1/breaches/spider/records?state=CA&org_type=MED&limit=50"

    Python Integration Example¶

    import requests
import time

BASE = "http://localhost:8001"

# Start extraction
requests.post(f"{BASE}/v1/breaches/spider/parallel", params={"strategy": "states"})

# Poll until complete
while True:
    status = requests.get(f"{BASE}/v1/breaches/spider/status").json()["data"]
    print(f"Status: {status['status']}, Records: {status.get('records_collected', 0)}")
    if status["status"] in ("completed", "failed"):
        break
    time.sleep(30)

# Get all healthcare breaches
records = requests.get(f"{BASE}/v1/breaches/spider/records", params={
    "org_type": "MED",
    "limit": 1000,
}).json()["data"]

print(f"Healthcare breaches: {records['total']}")
for r in records["records"][:5]:
    print(f"  {r['reported_date']} | {r['organization']} | {r['breach_type']}")
