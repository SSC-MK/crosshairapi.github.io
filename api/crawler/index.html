    Web Crawler¶

    Advanced web crawling with Human Behavior Simulation (HBS) using the Camoufox anti-detect browser engine.

    Overview¶

    The crawler endpoints provide full-featured web scraping with JavaScript rendering, bot-detection evasion, and structured content extraction. Every request is executed inside a real Firefox browser instance powered by Camoufox, which patches dozens of browser fingerprinting vectors.

    Human Behavior Simulation (HBS)

    HBS makes automated browsing indistinguishable from a real user by injecting:

    • Mouse movements -- realistic bezier-curve paths across the viewport
    • Scrolling -- variable-speed scroll events with pauses
    • Typing delays -- per-keystroke jitter that mimics human cadence
    • Session warmup -- visiting unrelated pages before the target to build a natural referrer chain

    Key Features¶

    Feature Description
    JavaScript rendering Full browser engine -- not headless Chrome stubs
    HBS anti-detection Mouse, scroll, typing, warmup, referrer spoofing, WebRTC blocking
    Crawl modes Single page, domain, subdomain, recursive
    Browsing modes fast, normal, human, stealth, amazon
    Bot-detection retry Exponential backoff when a challenge page is detected
    Content extraction Text, links, images, metadata
    Enhanced extraction Custom CSS / XPath / regex selectors, spaCy keyword extraction, language detection
    Screenshot capture Optional full-page screenshot per crawled URL
    Proxy support Datacenter and residential proxy pass-through

    Endpoints¶

    POST /v1/crawler/page -- Crawl Single Page¶

    Crawl a single page with full JavaScript rendering and optional enhanced extraction.

    Request Body¶

    {
  "url": "https://www.amazon.com/s?k=shoes",
  "browsing_mode": "normal",
  "timeout": 30000,
  "extract_content": true,
  "capture_screenshot": false,
  "warmup_browse": false,
  "selectors": [
    {"name": "title", "selector": "h1", "type": "css"},
    {"name": "prices", "selector": ".price", "type": "css", "multiple": true},
    {"name": "description", "selector": "//meta[@name='description']/@content", "type": "xpath"}
  ],
  "extract_keywords": true,
  "keyword_count": 10,
  "detect_language": true
}

    Browsing Modes¶

    Mode Delays Description
    fast 0.1 -- 0.3 s Minimal delays, fastest but most detectable
    normal 0.5 -- 1.5 s Good balance of speed and stealth
    human 1.5 -- 4 s Full HBS with mouse, scroll, typing
    stealth 3 -- 7 s Maximum anti-detection
    amazon 2 -- 5 s For heavily protected sites; includes warmup

    Enhanced Extraction Options¶

    Parameter Type Description
    selectors array Custom CSS / XPath / regex selectors for targeted extraction
    extract_keywords bool Extract keywords from page content (uses spaCy NLP)
    keyword_count int Number of keywords to extract (1-50, default: 10)
    detect_language bool Detect page language (uses langdetect)

    Selector Configuration¶

    {
  "name": "field_name",
  "selector": "h1.title",
  "type": "css",
  "attribute": "href",
  "multiple": true
}
    Field Required Description
    name yes Output field name
    selector yes CSS selector, XPath expression, or regex pattern
    type yes css, xpath, or regex
    attribute no Extract an attribute instead of inner text
    multiple no Return all matches instead of first only (default: false)

    Example¶

    curl -X POST http://localhost:8001/v1/crawler/page \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://www.amazon.com/s?k=shoes",
    "browsing_mode": "normal",
    "extract_content": true,
    "extract_keywords": true,
    "detect_language": true
  }'

    import requests

resp = requests.post("http://localhost:8001/v1/crawler/page", json={
    "url": "https://www.amazon.com/s?k=shoes",
    "browsing_mode": "normal",
    "extract_content": True,
    "extract_keywords": True,
    "detect_language": True,
})
data = resp.json()["data"]
print(data["title"], "-- links:", data["links_count"])

    POST /v1/crawler/crawl -- Multi-Page Crawl¶

    Start a multi-page crawl job with recursive link following.

    Request Body¶

    {
  "url": "https://example.com",
  "mode": "domain",
  "browsing_mode": "human",
  "max_depth": 2,
  "max_pages": 20,
  "min_delay": 1.0,
  "max_delay": 3.0,
  "extract_content": true,
  "capture_screenshots": false,
  "follow_external": false
}

    Crawl Modes¶

    Mode Scope
    single Only the starting URL
    domain Stay within the same domain (allows subdomains)
    subdomain Stay within the exact subdomain only
    recursive Follow all links (use with caution)

    Recursive Mode

    The recursive mode follows all outbound links regardless of domain. Always set max_pages and max_depth to avoid runaway crawls.

    Example¶

    curl -X POST http://localhost:8001/v1/crawler/crawl \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com",
    "mode": "domain",
    "browsing_mode": "human",
    "max_depth": 2,
    "max_pages": 20
  }'

    import requests

resp = requests.post("http://localhost:8001/v1/crawler/crawl", json={
    "url": "https://example.com",
    "mode": "domain",
    "browsing_mode": "human",
    "max_depth": 2,
    "max_pages": 20,
})
result = resp.json()["data"]
print(f"Crawled {result['pages_crawled']} pages in {result['elapsed_seconds']}s")

    Response¶

    {
  "success": true,
  "data": {
    "start_url": "https://example.com",
    "mode": "domain",
    "pages_crawled": 15,
    "pages_failed": 1,
    "pages_skipped": 4,
    "total_links_found": 342,
    "unique_domains": 1,
    "elapsed_seconds": 45.2,
    "stopped_reason": "max_pages",
    "results": ["..."]
  }
}

    Anti-Detection Features¶

    The crawler ships with multiple layers of anti-detection that work together to avoid bot fingerprinting.

    Layer Technique
    Browser engine Camoufox -- patched Firefox with fingerprint randomisation
    Mouse movements Bezier-curve paths with variable speed and overshoot
    Scrolling Human-like variable-speed scroll events
    Typing Per-keystroke jitter matching human cadence
    Session warmup Visits unrelated pages to build a natural referrer chain
    Referrer spoofing Constructs plausible referrer headers
    WebRTC blocking Prevents real-IP leaks through WebRTC
    Retry logic Exponential backoff on bot-detection challenge pages

    Using Proxies¶

    For datacenter IPs or heavily protected sites, add proxy configuration to any crawler request body:

    {
  "url": "https://www.amazon.com/s?k=shoes",
  "browsing_mode": "amazon",
  "proxy": {
    "server": "http://residential.proxy.com:8080",
    "username": "user",
    "password": "pass"
  }
}

    When Do You Need a Proxy?

    Residential IPs (home networks) typically do not need proxies. The proxy option is for users running from datacenter or cloud environments where the IP reputation is low.
