    Automation & Orchestration¶

    The Automation API provides a comprehensive suite of tools for building automated workflows: URL change monitors with webhook notifications, background job management, event-driven automation, conditional triggers, and a plugin system for extensibility.

    Monitors¶

    Monitors track URLs or search queries for changes over time, with optional webhook notifications for real-time alerts.

    Use Cases

    • Brand managers -- Monitor brand mentions and competitor activity
    • Security teams -- Track threat intelligence sources for changes
    • Researchers -- Watch topics for new developments
    • SEO teams -- Track ranking changes and competitor content

    List Monitors¶

    GET /v1/monitors

    List all configured monitors with their status and run history.

    Parameters:

    Name Type Default Description
    limit integer 20 Maximum monitors to return (1-100)

curl "http://localhost:8001/v1/monitors"

# List with pagination
curl "http://localhost:8001/v1/monitors?limit=50"

curl -s "http://localhost:8001/v1/monitors" \

    Monitor Object Fields:

    id string Unique monitor ID (12-char hex)

    Create Monitor¶

    POST /v1/monitors

    Create a new monitor to automatically track a URL or search query at a configurable interval.

    Request Body:

    Field Type Required Default Description
    url string Yes -- URL to monitor
    query string No -- Search query to monitor
    interval integer No 3600 Check interval in seconds (300-604800)
    name string No Auto-generated Monitor name
    notify_on_new boolean No true Notify on new results

    Headers:

    Name Description
    Idempotency-Key Prevent duplicate monitor creation 
    # Monitor a competitor's blog for new posts
curl -X POST http://localhost:8001/v1/monitors \
  -H "Content-Type: application/json" \
  -d '{"url": "https://competitor.com/blog", "name": "Competitor Blog", "interval": 3600}'

    # Monitor a GitHub releases page (idempotent)
curl -X POST http://localhost:8001/v1/monitors \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: github-react-releases" \
  -d '{"url": "https://github.com/facebook/react/releases", "name": "React Releases", "interval": 21600}'

    import requests

def create_brand_monitors(brand_name, check_interval=3600):
    sources = [
        ('Google News', f'https://news.google.com/search?q="{brand_name}"'),
        ('Twitter', f'https://twitter.com/search?q="{brand_name}"'),
        ('Reddit', f'https://reddit.com/search?q="{brand_name}"'),
    ]

    monitors = []
    for name, url in sources:
        response = requests.post(
            'http://localhost:8001/v1/monitors',
            json={
                'url': url,
                'name': f'{brand_name} - {name}',
                'interval': check_interval,
                'notify_on_new': True
            },
            headers={
                'Idempotency-Key': f'brand-{brand_name.lower()}-{name.lower()}'
            }
        )
        monitors.append(response.json()['data'])

    return monitors

    Configure Webhook¶

    POST /v1/monitors/{monitor_id}/webhook

    Attach a webhook to a monitor for real-time push notifications when changes are detected.

    HMAC Verification

    Use the secret field to enable HMAC-SHA256 signatures on webhook payloads. The signature is sent in the X-Signature header as sha256=<hex-digest>.

    Request Body:

    Field Type Default Description
    url string -- Webhook URL (required)
    events array ["change", "error"] Events to notify on
    secret string -- HMAC secret for payload signatures
    verify boolean true Verify webhook endpoint is reachable 
    curl -X POST http://localhost:8001/v1/monitors/abc123/webhook \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://hooks.slack.com/services/YOUR/SLACK/WEBHOOK",
    "events": ["change"],
    "verify": false
  }'

    curl -X POST http://localhost:8001/v1/monitors/abc123/webhook \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-server.com/webhooks/monitor",
    "events": ["change", "error"],
    "secret": "your-secret-key-32-chars-min",
    "verify": true
  }'

    import hmac
import hashlib
from flask import Flask, request, jsonify

app = Flask(__name__)
WEBHOOK_SECRET = 'your-secret-key-32-chars-min'

@app.route('/webhooks/monitor', methods=['POST'])
def handle_webhook():
    # Verify signature
    signature = request.headers.get('X-Signature', '')
    payload = request.get_data(as_text=True)

    expected = 'sha256=' + hmac.new(
        WEBHOOK_SECRET.encode(),
        payload.encode(),
        hashlib.sha256
    ).hexdigest()

    if not hmac.compare_digest(signature, expected):
        return jsonify({'error': 'Invalid signature'}), 401

    # Process event
    event = request.json
    if event['event'] == 'change':
        notify_team(f"Monitor {event['monitor_id']} detected changes!")

    return jsonify({'received': True})

    Jobs¶

    Jobs represent asynchronous background operations such as batch content extraction. Jobs auto-expire after 24 hours.

    Job Lifecycle

    Jobs transition through states: pending -> running -> completed | failed | cancelled

    Get Job Status¶

    GET /v1/jobs/{job_id}

    Poll for the status of an asynchronous batch operation.

    curl "http://localhost:8001/v1/jobs/e31f008a-303f-4ec2-a994-e26cf4ce677a"

    JOB_ID="e31f008a-303f-4ec2-a994-e26cf4ce677a"
while true; do
  STATUS=$(curl -s "http://localhost:8001/v1/jobs/$JOB_ID" | jq -r '.data.status')
  echo "Status: $STATUS"
  [ "$STATUS" = "completed" ] && break
  [ "$STATUS" = "failed" ] && exit 1
  sleep 5
done
curl "http://localhost:8001/v1/jobs/$JOB_ID" | jq '.data.results'

    import requests
import time

def wait_for_job(job_id, timeout=300, poll_interval=5):
    start = time.time()
    while time.time() - start < timeout:
        response = requests.get(f'http://localhost:8001/v1/jobs/{job_id}')
        data = response.json()['data']

        if data['status'] == 'completed':
            return data['results']
        elif data['status'] == 'failed':
            raise Exception(f"Job failed: {data.get('error')}")
        elif data['status'] == 'cancelled':
            raise Exception("Job was cancelled")

        print(f"Progress: {data['completed']}/{data['total']}")
        time.sleep(poll_interval)

    raise TimeoutError(f"Job {job_id} did not complete in {timeout}s")

    Cancel Job¶

    DELETE /v1/jobs/{job_id}

    Cancel a pending or running job. Already-completed jobs cannot be cancelled.

    # Cancel a single job
curl -X DELETE "http://localhost:8001/v1/jobs/e31f008a-303f-4ec2-a994-e26cf4ce677a"

    for job in $(curl -s "http://localhost:8001/v1/jobs" \
  | jq -r '.data.jobs[] | select(.status=="pending") | .job_id'); do
  curl -X DELETE "http://localhost:8001/v1/jobs/$job"
done

    Events¶

    Event-driven automation system for publishing and consuming custom events.

    List Events¶

    GET /v1/events

    List recent events in the system.

    curl "http://localhost:8001/v1/events"

    Emit Event¶

    POST /v1/events/emit

    Publish a custom event into the event bus.

    Request Body:

    {
  "event_type": "custom.alert",
  "data": {"message": "Test alert"}
}

    curl -X POST http://localhost:8001/v1/events/emit \
  -H "Content-Type: application/json" \
  -d '{"event_type": "custom.alert", "data": {"message": "Test alert"}}'

    Event Statistics¶

    GET /v1/events/stats

    Retrieve aggregated event statistics.

    curl "http://localhost:8001/v1/events/stats"

    Triggers¶

    Configure automation rules that execute actions when specific event conditions are met.

    How Triggers Work

    A trigger watches for events of a given event_type, evaluates its conditions against the event data, and executes actions (such as webhooks) when conditions match.

    List Triggers¶

    GET /v1/triggers

    List all configured triggers.

    curl "http://localhost:8001/v1/triggers"

    Create Trigger¶

    POST /v1/triggers

    Create a new automation trigger.

    Request Body:

    {
  "name": "Alert on malware",
  "event_type": "scan.completed",
  "conditions": {"verdict": "malicious"},
  "actions": [{"type": "webhook", "url": "https://..."}]
}

    curl -X POST http://localhost:8001/v1/triggers \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Alert on malware",
    "event_type": "scan.completed",
    "conditions": {"verdict": "malicious"},
    "actions": [{"type": "webhook", "url": "https://your-server.com/alerts"}]
  }'

    Get Trigger Details¶

    GET /v1/triggers/{trigger_id}

    Retrieve configuration details for a specific trigger.

    curl "http://localhost:8001/v1/triggers/trg_abc123"

    Update Trigger¶

    PUT /v1/triggers/{trigger_id}

    Update an existing trigger's configuration.

    curl -X PUT http://localhost:8001/v1/triggers/trg_abc123 \
  -H "Content-Type: application/json" \
  -d '{"name": "Updated trigger name"}'

    Delete Trigger¶

    DELETE /v1/triggers/{trigger_id}

    Remove a trigger.

    curl -X DELETE "http://localhost:8001/v1/triggers/trg_abc123"

    Test Trigger¶

    POST /v1/triggers/test

    Test trigger conditions against sample event data without actually firing actions.

    curl -X POST http://localhost:8001/v1/triggers/test \
  -H "Content-Type: application/json" \
  -d '{"event_type": "scan.completed", "data": {"verdict": "malicious"}}'

    Plugins¶

    Extensible plugin system built on the Pluggy framework. Plugins can extend API functionality at runtime.

    List Plugins¶

    GET /v1/plugins

    List all loaded plugins and their status.

    curl "http://localhost:8001/v1/plugins"

    Load Plugin¶

    POST /v1/plugins/load

    Load a new plugin into the system.

    curl -X POST http://localhost:8001/v1/plugins/load \
  -H "Content-Type: application/json" \
  -d '{"path": "/path/to/plugin.py"}'

    Reload All Plugins¶

    POST /v1/plugins/reload

    Reload all currently loaded plugins.

    curl -X POST http://localhost:8001/v1/plugins/reload

    Get Plugin Details¶

    GET /v1/plugins/{name}

    Get detailed information about a specific plugin.

    curl "http://localhost:8001/v1/plugins/my-plugin"

    Enable Plugin¶

    POST /v1/plugins/{name}/enable

    Enable a disabled plugin.

    curl -X POST http://localhost:8001/v1/plugins/my-plugin/enable

    Disable Plugin¶

    POST /v1/plugins/{name}/disable

    Disable a plugin without unloading it.

    curl -X POST http://localhost:8001/v1/plugins/my-plugin/disable

    Reload Plugin¶

    POST /v1/plugins/{name}/reload

    Reload a specific plugin to pick up code changes.

    curl -X POST http://localhost:8001/v1/plugins/my-plugin/reload

    Unload Plugin¶

    DELETE /v1/plugins/{name}

    Completely unload a plugin from the system.

    curl -X DELETE "http://localhost:8001/v1/plugins/my-plugin"
