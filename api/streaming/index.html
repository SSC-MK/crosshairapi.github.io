    Streaming Endpoints¶

    The Streaming API uses Server-Sent Events (SSE) to deliver search results in real time as they arrive from upstream engines. Instead of waiting for all engines to respond before returning a single JSON payload, SSE streams individual results the moment they are available -- reducing perceived latency from seconds to milliseconds.

    SSE Overview¶

    What are Server-Sent Events?

    SSE is a standard HTTP protocol (part of the HTML5 spec) that allows a server to push events to the client over a single, long-lived HTTP connection. Unlike WebSockets, SSE is unidirectional (server to client), uses plain HTTP, and works through proxies and firewalls without special configuration.

    How It Works¶

    1. The client opens a GET request with Accept: text/event-stream
    2. The server holds the connection open and sends events as they become available
    3. Each event has a type (event:) and payload (data:)
    4. The client closes the connection when the complete event arrives

    SSE Event Format¶

    Each event in the stream follows this structure:

    event: <event_type>
data: <json_payload>

    Events are separated by a blank line. The data: field contains a JSON object that varies by event type.

    Endpoints¶

    GET /v1/stream/search¶

    Server-Sent Events stream for real-time search results.

    Best For

    Building responsive UIs that display results as they arrive.

    Target Audience

    • Frontend developers -- Building real-time search interfaces
    • UX designers -- Creating progressive loading experiences
    • Performance-focused teams -- Reducing perceived latency
    • Dashboard builders -- Live updating search widgets

    Why Use This:

    • Results appear immediately as they are fetched (no waiting for all engines)
    • Reduces perceived latency -- users see first result in ~500ms vs 2-3s
    • Perfect for "infinite scroll" or progressive loading UIs
    • Includes completion event with total count and timing
    • Standard SSE protocol works with all modern browsers

    Parameters:

    Name Type Required Default Description
    q string Yes -- Search query
    engines string No google,brave,duckduckgo Comma-separated engines
    limit integer No 10 Maximum results (1-50)

    Event Types¶

    Event Description
    start Stream started -- contains query info and request ID
    result Individual search result object
    error Error occurred during search
    complete Stream finished -- contains summary with total count and timing

    Example Request¶

    curl -N "http://localhost:8001/v1/stream/search?q=python&limit=5"

    Example SSE Response¶

    event: start
data: {"query": "python", "request_id": "uuid", "engines": ["google"], "limit": 5}

event: result
data: {"position": 1, "title": "Python.org", "url": "https://python.org", ...}

event: result
data: {"position": 2, "title": "Python Tutorial", "url": "https://...", ...}

event: complete
data: {"total": 5, "engines_used": ["google", "brave"], "processing_time_ms": 2500}

    Client Examples¶

    const eventSource = new EventSource(
  'http://localhost:8001/v1/stream/search?q=python&limit=10'
);

eventSource.addEventListener('result', (e) => {
  const result = JSON.parse(e.data);
  console.log('New result:', result.title);
});

eventSource.addEventListener('complete', (e) => {
  const summary = JSON.parse(e.data);
  console.log(`Done: ${summary.total} results in ${summary.processing_time_ms}ms`);
  eventSource.close();
});

eventSource.addEventListener('error', (e) => {
  console.error('Stream error');
  eventSource.close();
});

    import { useEffect, useState } from 'react';

function LiveSearch({ query }) {
  const [results, setResults] = useState([]);
  const [status, setStatus] = useState('idle');

  useEffect(() => {
    if (!query) return;

    setResults([]);
    setStatus('searching');

    const eventSource = new EventSource(
      `http://localhost:8001/v1/stream/search?q=${encodeURIComponent(query)}&limit=20`
    );

    eventSource.addEventListener('result', (e) => {
      const result = JSON.parse(e.data);
      setResults(prev => [...prev, result]);
    });

    eventSource.addEventListener('complete', (e) => {
      setStatus('complete');
      eventSource.close();
    });

    eventSource.addEventListener('error', (e) => {
      setStatus('error');
      eventSource.close();
    });

    return () => eventSource.close();
  }, [query]);

  return (
    <div>
      {results.map((r, i) => <SearchResult key={i} {...r} />)}
      {status === 'searching' && <Spinner />}
    </div>
  );
}

    import requests
import sseclient
import json

url = 'http://localhost:8001/v1/stream/search?q=python+programming&limit=10'
response = requests.get(url, stream=True)
client = sseclient.SSEClient(response)

for event in client.events():
    if event.event == 'result':
        result = json.loads(event.data)
        print(f"[{result['position']}] {result['title']}")
        print(f"    {result['url']}")
    elif event.event == 'complete':
        summary = json.loads(event.data)
        print(f"\nDone: {summary['total']} results in {summary['processing_time_ms']}ms")
        break
    elif event.event == 'error':
        print(f"Error: {event.data}")
        break

    # Stream results to terminal
curl -N "http://localhost:8001/v1/stream/search?q=python+programming&limit=10"

# Watch results arrive in real-time
curl -N "http://localhost:8001/v1/stream/search?q=breaking+news" | while read line; do
  echo "$line"
done

    GET /v1/stream/multi-search¶

    Stream results from multiple search queries in parallel using SSE.

    Best For

    Running multiple related searches simultaneously and streaming all results.

    Parameters:

    Name Type Required Default Description
    q string Yes -- Comma-separated search queries
    engines string No google,brave,duckduckgo Search engines
    limit integer No 10 Results per query (1-50)

    Event Types¶

    Event Description
    query_start Indicates a new query is starting (includes query text)
    result Individual search result (includes query identifier)
    query_complete A single query finished with per-query stats
    complete All queries finished

    Example Request¶

    # Search for multiple topics simultaneously
curl -N "http://localhost:8001/v1/stream/multi-search?q=python+tutorial,javascript+basics,rust+programming&limit=5"

    Use Cases¶

    Use Cases

    • Comparison dashboards -- Compare results across multiple search terms side by side
    • Multi-topic research -- Research several related topics in a single request
    • A/B query testing -- Compare how different phrasings return different results
