    Content Endpoints¶

    The Content API provides powerful tools for extracting, transforming, and anonymizing web content. From converting web pages to clean text or markdown, to generating screenshots, discovering CSS/XPath selectors, and stripping PII from documents -- these endpoints form a complete content processing pipeline.

    Capabilities Overview¶

    Capability Endpoint Description
    Text Extraction POST /v1/content/extract Extract clean text from web pages and PDFs
    Batch Extraction POST /v1/content/extract/batch Queue bulk URL extraction as async jobs
    Screenshots GET /v1/content/screenshot Generate visual previews via multiple services
    Selector Discovery GET /v1/content/selectors Extract XPath, CSS, and Playwright selectors
    Markdown Conversion GET /v1/content/markdown Convert web pages to clean markdown
    PII Anonymization POST /v1/content/anonymize Detect and anonymize personally identifiable information
    URL Anonymization POST /v1/content/anonymize/url Extract content from a URL and anonymize PII in one step
    PII Entity Types GET /v1/content/pii/entities List all supported PII entity types

    POST /v1/content/extract¶

    Extract text content from a URL. Removes HTML boilerplate and returns the main content along with NLP-extracted entities and an optional LLM summary.

    Best For

    Converting web pages to clean text for analysis, storage, or processing.

    Target Audience

    • Data engineers -- Building content pipelines and datasets
    • Researchers -- Extracting article text for analysis
    • AI/ML teams -- Preparing training data from web sources
    • Content aggregators -- Building read-it-later or RSS-like services
    • SEO analysts -- Extracting and analyzing competitor content

    Key features:

    • Removes HTML boilerplate and extracts main content
    • Handles PDFs via MinerU integration
    • Automatic NLP entity extraction (people, orgs, locations)
    • Optional LLM summary for quick understanding
    • SSRF-protected (blocks internal network access)

    Parameters¶

    Request Body:

    Field Type Required Description
    url string Yes URL to extract (web page or PDF)
    options object No Additional extraction options

    Query Parameters:

    Name Type Default Description
    llm boolean false Add LLM-generated summary

    Examples¶

    curl -X POST http://localhost:8001/v1/content/extract \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com"}'

    curl -X POST "http://localhost:8001/v1/content/extract?llm=true" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://arxiv.org/abs/2301.07041"}'

    curl -X POST http://localhost:8001/v1/content/extract \
  -H "Content-Type: application/json" \
  -d '{"url": "https://example.com/report.pdf"}'

    import requests

urls = [
    "https://blog.example.com/post-1",
    "https://blog.example.com/post-2",
]

for url in urls:
    response = requests.post(
        "http://localhost:8001/v1/content/extract",
        json={"url": url}
    )
    data = response.json()["data"]

    save_to_database({
        "url": url,
        "content": data["content"],
        "entities": data["entities"],
        "extracted_at": data["meta"]["timestamp"]
    })

    3. Indicates whether the source URL was a PDF. When true, extraction was handled by the MinerU PDF processing service instead of HTML parsing.

    llm_summary string LLM-generated summary (only if llm=true)

    POST /v1/content/extract/batch¶

    Queue batch URL extraction as an async job. Returns immediately with a job ID for status tracking. Supports webhook notifications on completion.

    Best For

    Processing many URLs without blocking -- ideal for data pipelines.

    Target Audience

    • Data engineers -- Building ETL pipelines for web content
    • Researchers -- Extracting content from large URL lists
    • Archivists -- Bulk preserving web content
    • ML engineers -- Preparing training datasets from web sources

    Key features:

    • Non-blocking: returns immediately with job ID
    • Parallel processing with automatic rate limiting (semaphore-controlled)
    • Webhook notifications when job completes
    • Idempotency keys prevent duplicate processing
    • HMAC-signed webhooks for security

    Parameters¶

    Request Body:

    Field Type Required Description
    urls array Yes List of URLs to extract
    callback_url string No Webhook URL for completion notification
    callback_secret string No HMAC secret for webhook signature verification

    Headers:

    Name Description
    Idempotency-Key Prevent duplicate job creation

    Examples¶

    curl -X POST http://localhost:8001/v1/content/extract/batch \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: batch-job-123" \
  -d '{"urls": ["https://example.com", "https://example.org"]}'

    curl -X POST http://localhost:8001/v1/content/extract/batch \
  -H "Content-Type: application/json" \
  -d '{
    "urls": ["https://example.com", "https://example.org"],
    "callback_url": "https://your-server.com/webhook",
    "callback_secret": "your-secret-key"
  }'

    JOB_ID="e31f008a-303f-4ec2-a994-e26cf4ce677a"
curl "http://localhost:8001/v1/jobs/$JOB_ID"

    import requests
import time

# Submit batch
response = requests.post(
    "http://localhost:8001/v1/content/extract/batch",
    json={"urls": research_urls},  # List of 100+ URLs
    headers={"Idempotency-Key": f"research-batch-{session_id}"}
)

job_id = response.json()["data"]["job_id"]

# Poll for completion (or use webhook)
while True:
    status = requests.get(
        f"http://localhost:8001/v1/jobs/{job_id}"
    ).json()
    if status["data"]["status"] == "completed":
        results = status["data"]["results"]
        break
    time.sleep(5)

    GET /v1/content/screenshot¶

    Generate screenshot URLs for a webpage using multiple screenshot services (thum.io, screenshotmachine, microlink). Returns URLs directly -- no image storage required on your server.

    Best For

    Generating visual previews of web pages for thumbnails, archives, or monitoring.

    Target Audience

    • Product managers -- Creating visual documentation of web pages
    • QA engineers -- Capturing visual state for bug reports
    • Archivists -- Preserving visual snapshots of websites
    • Marketers -- Creating social media previews
    • Developers -- Building link preview features

    Parameters¶

    Name Type Default Description
    url string -- URL to screenshot (required)
    width integer 1280 Screenshot width in pixels
    height integer 800 Screenshot height in pixels

    Examples¶

    curl "http://localhost:8001/v1/content/screenshot?url=https://example.com"

    curl "http://localhost:8001/v1/content/screenshot?url=https://example.com&width=1920&height=1080"

    curl "http://localhost:8001/v1/content/screenshot?url=https://example.com&width=375&height=812"

    async function getLinkPreview(url) {
  const response = await fetch(
    `http://localhost:8001/v1/content/screenshot?url=${encodeURIComponent(url)}`
  );
  const data = await response.json();

  // Use thum.io by default, fall back to others if rate limited
  return {
    thumbnail: data.data.screenshots.thum_io,
    fallback: data.data.screenshots.microlink
  };
}

    GET /v1/content/selectors¶

    Extract XPath, CSS, and Playwright selectors from a webpage using browser automation. Supports semantic search to find elements by description (e.g., "login button", "search box").

    Best For

    Building web scrapers, automation scripts, and testing frameworks.

    Target Audience

    • Automation engineers -- Finding reliable selectors for Playwright/Selenium
    • QA engineers -- Generating selectors for test automation
    • Data engineers -- Building web scraping pipelines
    • RPA developers -- Creating robotic process automation workflows
    • Web scraping teams -- Identifying stable element locators

    Key features:

    • Semantic search: Find elements by description ("login button", "search box")
    • Quality ratings: Know if a selector is stable (GOOD) or fragile (POOR)
    • Multiple selector types: XPath, CSS, and Playwright locators
    • Bounding boxes: Know exact element positions for click automation
    • Browser-rendered: Sees JavaScript-rendered content (not just raw HTML)

    Parameters¶

    Name Type Default Description
    url string -- URL to analyze (required)
    q string -- Semantic search for elements (e.g., "login button")
    full boolean false Include full absolute XPath/CSS
    limit integer 20 Maximum elements to return (1-100)

    Examples¶

    curl "http://localhost:8001/v1/content/selectors?url=https://example.com&q=link&limit=5"

    curl "http://localhost:8001/v1/content/selectors?url=https://example.com&q=input+field"

    curl "http://localhost:8001/v1/content/selectors?url=https://example.com&q=navigation&full=true"

    import requests
from playwright.sync_api import sync_playwright

# Find the best selector for the "Add to Cart" button
response = requests.get(
    "http://localhost:8001/v1/content/selectors",
    params={
        "url": "https://shop.example.com/product/123",
        "q": "add to cart button"
    }
)

matches = response.json()["data"]["matches"]

# Use the highest quality selector
best_selector = None
for match in matches:
    for sel in match["selectors"]:
        if sel["quality"] == "GOOD":
            best_selector = sel["selector"]
            break

# Use in Playwright
with sync_playwright() as p:
    browser = p.chromium.launch()
    page = browser.new_page()
    page.goto("https://shop.example.com/product/123")
    page.click(best_selector)

    # Generate selectors for a form page
curl "http://localhost:8001/v1/content/selectors?url=https://app.example.com/signup&q=form" | \
  jq '.data.matches[] | {element: .name, xpath: .xpath, quality: .xpath_quality}'

    GET /v1/content/markdown¶

    Convert a webpage to clean markdown. Uses the reader-lm model for intelligent extraction that removes ads, navigation, and clutter while preserving document structure.

    Best For

    Preparing web content for LLMs, documentation, or offline reading.

    Target Audience

    • AI/ML engineers -- Preparing clean text for LLM context windows
    • Documentation teams -- Importing web content into docs
    • Researchers -- Creating clean, portable versions of articles
    • Content migrators -- Moving content between platforms
    • Read-it-later apps -- Building offline reading features

    Key features:

    • Removes ads, navigation, and clutter -- keeps main content
    • Preserves document structure (headings, lists, code blocks)
    • Optional link/image inclusion for different use cases
    • Uses reader-lm model for intelligent extraction
    • Output is LLM-ready (clean context for RAG, summarization)

    Parameters¶

    Name Type Default Description
    url string -- URL to convert (required)
    include_links boolean true Include hyperlinks in output
    include_images boolean false Include image references

    Examples¶

    curl "http://localhost:8001/v1/content/markdown?url=https://example.com"

    curl "http://localhost:8001/v1/content/markdown?url=https://docs.example.com/guide&include_links=false"

    curl "http://localhost:8001/v1/content/markdown?url=https://example.com&include_images=true"

    import requests

def prepare_for_rag(urls):
    documents = []
    for url in urls:
        response = requests.get(
            "http://localhost:8001/v1/content/markdown",
            params={
                "url": url,
                "include_links": False,   # Cleaner for embeddings
                "include_images": False
            }
        )
        data = response.json()["data"]
        documents.append({
            "url": url,
            "title": data["title"],
            "content": data["markdown"],
            "char_count": data["content_length"]
        })
    return documents

docs = prepare_for_rag(article_urls)

    POST /v1/content/anonymize¶

    Anonymize PII (Personally Identifiable Information) in text using Microsoft Presidio. Detects and anonymizes names, emails, phone numbers, credit cards, and many more entity types.

    Best For

    Data privacy compliance, anonymizing logs, redacting sensitive data before sharing or storage.

    Parameters¶

    Name Type Required Default Description
    text string Yes -- Text to anonymize
    entities array No all Entity types to detect (detects all if omitted)
    default_operator string No replace Anonymization operator: replace, redact, mask, hash
    score_threshold float No 0.5 Confidence threshold 0-1
    return_entities boolean No true Include detected entities in response

    Anonymization Operators¶

    Operator Behavior Example Output
    replace Replace with entity type label <PERSON>
    redact Remove completely (empty)
    mask Partially mask ****@email.com
    hash SHA256 hash a1b2c3d4e5f6...

    Examples¶

    curl -X POST http://localhost:8001/v1/content/anonymize \
  -H "Content-Type: application/json" \
  -d '{"text": "Contact John at john@example.com"}'

    curl -X POST http://localhost:8001/v1/content/anonymize \
  -H "Content-Type: application/json" \
  -d '{
    "text": "John Smith'\''s email is john@example.com and phone is 555-123-4567",
    "entities": ["PERSON", "EMAIL_ADDRESS", "PHONE_NUMBER"],
    "default_operator": "replace",
    "score_threshold": 0.5,
    "return_entities": true
  }'

    curl -X POST http://localhost:8001/v1/content/anonymize \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Contact John at john@example.com",
    "default_operator": "mask"
  }'

    Response¶

    {
  "success": true,
  "data": {
    "text": "Contact <PERSON> at <EMAIL_ADDRESS>",
    "original_length": 32,
    "anonymized_length": 37,
    "entities_detected": 2,
    "entity_counts": {"PERSON": 1, "EMAIL_ADDRESS": 1},
    "entities": [
      {"entity_type": "PERSON", "text": "John", "start": 8, "end": 12, "score": 0.85},
      {"entity_type": "EMAIL_ADDRESS", "text": "john@example.com", "start": 16, "end": 32, "score": 0.95}
    ]
  }
}

    Response Fields¶

    Field Type Description
    text string Anonymized text
    original_length integer Character count of original text
    anonymized_length integer Character count after anonymization
    entities_detected integer Total number of PII entities found
    entity_counts object Count of each entity type detected
    entities array Detailed list of detected entities with positions and confidence scores

    POST /v1/content/anonymize/url¶

    Extract content from a URL and anonymize PII in a single step. Combines the extraction and anonymization pipelines.

    Parameters¶

    Name Type Required Default Description
    url string Yes -- URL to extract and anonymize
    default_operator string No replace Anonymization operator
    score_threshold float No 0.5 Confidence threshold 0-1

    Example¶

    curl -X POST http://localhost:8001/v1/content/anonymize/url \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/page-with-pii",
    "default_operator": "replace",
    "score_threshold": 0.5
  }'
