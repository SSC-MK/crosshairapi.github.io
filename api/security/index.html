    Security

    Security Testing¶

    The Security API provides fuzzing, vulnerability scanning, and CVE intelligence tools for authorized security assessments.

    Authorization Required

    Only use these endpoints against systems you own or have explicit written permission to test. Unauthorized security testing is illegal in most jurisdictions.

    Directory Fuzzing¶

    Discover hidden directories, files, and paths on a target web server using wordlist-based fuzzing.

    List Wordlists¶

    GET /v1/security/fuzz/wordlists

    List available built-in wordlists for directory and file fuzzing.

    curl http://localhost:8001/v1/security/fuzz/wordlists
    Response 
    {
  "success": true,
  "data": {
    "wordlists": [
      {"name": "common", "description": "Common directories and files", "count": 50, "category": "directories"},
      {"name": "dirs", "description": "Directory names only", "count": 35, "category": "directories"},
      {"name": "files", "description": "Common file names", "count": 40, "category": "files"},
      {"name": "api", "description": "Common API endpoints", "count": 30, "category": "api"}
    ]
  }
}

    Available Wordlists:

    Name Category Count Description
    common directories 50 Common directories and files
    dirs directories 35 Directory names only
    files files 40 Common file names
    api api 30 Common API endpoints

    List Payloads¶

    GET /v1/security/fuzz/payloads

    List available payload sets for parameter fuzzing.

    curl http://localhost:8001/v1/security/fuzz/payloads
    Response 
    {
  "success": true,
  "data": {
    "payloads": [
      {"name": "xss", "description": "XSS test payloads", "count": 10, "category": "xss"},
      {"name": "sqli", "description": "SQL injection payloads", "count": 15, "category": "sqli"},
      {"name": "lfi", "description": "LFI test payloads", "count": 10, "category": "lfi"},
      {"name": "traversal", "description": "Path traversal payloads", "count": 10, "category": "traversal"}
    ]
  }
}

    Available Payload Sets:

    Name Category Count Description
    xss xss 10 XSS test payloads
    sqli sqli 15 SQL injection payloads
    lfi lfi 10 LFI test payloads
    traversal traversal 10 Path traversal payloads

    Fuzz Directories¶

    POST /v1/security/fuzz/directories

    Perform wfuzz-style directory and path discovery on a target.

    Parameters:

    Name Type Default Description
    target string -- Target base URL (required)
    wordlist string -- Built-in wordlist name
    paths array -- Custom paths to test
    extensions array -- File extensions to append (e.g., .php, .bak)
    limit integer 100 Maximum requests
    rate_limit float 10 Requests per second
    timeout float 10 Per-request timeout in seconds 
    curl -X POST http://localhost:8001/v1/security/fuzz/directories \
  -H "Content-Type: application/json" \
  -d '{
    "target": "http://example.com",
    "wordlist": "common",
    "extensions": [".php", ".bak"],
    "limit": 100,
    "rate_limit": 10
  }'

    curl -X POST http://localhost:8001/v1/security/fuzz/directories \
  -H "Content-Type: application/json" \
  -d '{
    "target": "http://example.com",
    "paths": ["admin", "backup", "config", ".git", ".env"],
    "extensions": [".php", ".bak", ".old"]
  }'
    Response 
    {
  "success": true,
  "data": {
    "target": "http://example.com",
    "results": [
      {"url": "http://example.com/admin", "status_code": 200, "content_length": 1234}
    ],
    "found": 1,
    "tested": 50,
    "elapsed_ms": 5200
  }
}

    Parameter Fuzzing¶

    Fuzz Parameters¶

    POST /v1/security/fuzz/params

    Fuzz URL or form parameters with payload sets to detect injection vulnerabilities.

    Request Body:

    Name Type Description
    target string Target URL (required)
    method string HTTP method: GET or POST
    params object Parameters with FUZZ placeholder
    payloads string Payload set name (e.g., xss, sqli)
    limit integer Maximum requests 
    curl -X POST http://localhost:8001/v1/security/fuzz/params \
  -H "Content-Type: application/json" \
  -d '{
    "target": "http://example.com/search",
    "method": "GET",
    "params": {"q": "FUZZ"},
    "payloads": "xss",
    "limit": 50
  }'

    curl -X POST http://localhost:8001/v1/security/fuzz/params \
  -H "Content-Type: application/json" \
  -d '{
    "target": "http://example.com/search",
    "method": "GET",
    "params": {"q": "FUZZ"},
    "payloads": "sqli",
    "limit": 50
  }'

    Vulnerability Scanning¶

    Automated vulnerability detection covering security headers, XSS, SQL injection, and local file inclusion.

    Scan Security Headers¶

    GET /v1/security/scan/headers

    Analyze HTTP security headers for a URL and produce a security grade.

    Parameters:

    Name Type Description
    url string URL to scan (required) 
    curl "http://localhost:8001/v1/security/scan/headers?url=https://example.com"
    Response 
    {
  "success": true,
  "data": {
    "url": "https://example.com",
    "headers": {
      "Strict-Transport-Security": {"value": "max-age=31536000", "present": true}
    },
    "missing": ["Content-Security-Policy", "X-Frame-Options"],
    "issues": ["Missing critical header: Content-Security-Policy"],
    "score": 65,
    "grade": "C"
  }
}

    Scan for XSS¶

    POST /v1/security/scan/xss

    Scan a URL for Cross-Site Scripting vulnerabilities.

    Request Body:

    Field Type Description
    url string Target URL (required)
    params object URL parameters to test
    depth integer Scan depth level 
    curl -X POST http://localhost:8001/v1/security/scan/xss \
  -H "Content-Type: application/json" \
  -d '{"url": "http://example.com/search", "params": {"q": "test"}, "depth": 1}'

    Scan for SQL Injection¶

    POST /v1/security/scan/sqli

    Scan a URL for SQL Injection vulnerabilities.

    Request Body:

    Field Type Description
    url string Target URL (required)
    params object URL parameters to test
    depth integer Scan depth level 
    curl -X POST http://localhost:8001/v1/security/scan/sqli \
  -H "Content-Type: application/json" \
  -d '{"url": "http://example.com/item", "params": {"id": "1"}, "depth": 1}'

    Scan for LFI¶

    POST /v1/security/scan/lfi

    Scan a URL for Local File Inclusion vulnerabilities.

    Request Body:

    Field Type Description
    url string Target URL (required)
    params object URL parameters to test 
    curl -X POST http://localhost:8001/v1/security/scan/lfi \
  -H "Content-Type: application/json" \
  -d '{"url": "http://example.com/page", "params": {"file": "index.php"}}'

    Full Vulnerability Scan¶

    POST /v1/security/scan

    Run a comprehensive vulnerability scan combining all available checks.

    Request Body:

    Field Type Default Description
    url string -- Target URL (required)
    params object -- Parameters to test
    checks array All checks Checks to run: headers, xss, sqli, lfi
    depth integer 1 Scan depth level 
    curl -X POST http://localhost:8001/v1/security/scan \
  -H "Content-Type: application/json" \
  -d '{
    "url": "http://example.com",
    "params": {"q": "test"},
    "checks": ["headers", "xss", "sqli", "lfi"],
    "depth": 1
  }'

    curl -X POST http://localhost:8001/v1/security/scan \
  -H "Content-Type: application/json" \
  -d '{
    "url": "http://example.com",
    "checks": ["headers"]
  }'
    Response 
    {
  "success": true,
  "data": {
    "url": "http://example.com",
    "vulnerable": true,
    "summary": {"high": 1, "medium": 2, "info": 3},
    "findings": [
      {
        "type": "xss",
        "severity": "high",
        "url": "http://example.com/search?q=<script>",
        "parameter": "q",
        "description": "Potential XSS - payload reflected",
        "remediation": "Sanitize and encode user input"
      }
    ],
    "checks_performed": ["headers", "xss", "sqli", "lfi"],
    "tested": 45
  }
}

    Severity Levels

    Findings are categorized by severity:

    Severity Description
    high Exploitable vulnerability requiring immediate attention
    medium Potential vulnerability or configuration weakness
    info Informational finding or best practice recommendation

    Port Scanning & CVE Intelligence¶

    Network-level vulnerability assessment with port scanning and CVE database lookup.

    Check Availability¶

    GET /v1/vuln/check

    Check if the vulnerability scanning subsystem is available and operational.

    curl http://localhost:8001/v1/vuln/check

    Scan Target¶

    POST /v1/vuln/scan

    Perform a vulnerability scan on a target host, checking specified ports for known vulnerabilities.

    Request Body:

    Field Type Description
    target string Target hostname or IP (required)
    ports array List of specific ports to scan
    timeout integer Scan timeout in seconds 
    curl -X POST http://localhost:8001/v1/vuln/scan \
  -H "Content-Type: application/json" \
  -d '{"target": "example.com", "ports": [80, 443, 8080], "timeout": 30}'

    Scan Ports¶

    POST /v1/vuln/ports

    Perform a port scan on a target to discover open services.

    Request Body:

    Field Type Description
    target string Target hostname or IP (required)
    ports string Port range (e.g., 1-1000, 80,443,8080) 
    curl -X POST http://localhost:8001/v1/vuln/ports \
  -H "Content-Type: application/json" \
  -d '{"target": "example.com", "ports": "1-1000"}'

    Search CVE Database¶

    GET /v1/vuln/cve/search

    Search the CVE (Common Vulnerabilities and Exposures) database.

    Parameters:

    Name Type Description
    query string Free-text search query
    product string Filter by product name
    severity string Filter by severity: CRITICAL, HIGH, MEDIUM, LOW
    limit integer Maximum results to return 
    curl "http://localhost:8001/v1/vuln/cve/search?product=apache&severity=CRITICAL&limit=10"

    curl "http://localhost:8001/v1/vuln/cve/search?query=log4j&severity=CRITICAL"

    Get CVE Details¶

    GET /v1/vuln/cve/{cve_id}

    Retrieve detailed information about a specific CVE entry.

    curl http://localhost:8001/v1/vuln/cve/CVE-2021-44228
